# 鳥羽高校HP技術監査・刷新スキーム報告書

## 1. 現状の技術スタックと制約
- **CMS**: Movable Type (京都みらいネット提供の共有環境)
- **ドメイン**: `kyoto-be.ne.jp` サブドメイン
- **制約事項**:
    - **独自ドメイン**: 移行は技術的に可能だが、教育ネットワーク事務局の承認が必要。
    - **JavaScript実行**: セキュリティ実施手順により、外部スクリプト（SNS埋め込み用ウィジェット等）の自由な挿入が制限されている可能性がある。
    - **Movable Typeの提供形態**: クラウド型（MovableType.net系）か、オンプレミス・インストール型かによりプラグイン導入の自由度が異なる。

## 2. Instagram連携の実現パス：3つの案

### 案A：ネイティブ・ウィジェット埋め込み（難易度：低）
- **手法**: Behold や Elfsight 等の外部サービスで埋め込みコードを生成し、MTのテンプレートに貼り付ける。
- **課題**: セキュリティ規定で外部JSがブロックされる可能性がある。

### 案B：Instagram投稿の「記事化」プラグイン（難易度：中）
- **手法**: `InstaPost` 等のプラグインを活用し、Instagramに投稿した内容を自動的にMTの記事として取り込む。
- **メリット**: HP内のコンテンツとして蓄積されるため、SEOに強く、セキュリティ制限に引っかからない。

### 案C：HPトップへの「追従バナー」＋リンク（難易度：極低）
- **手法**: インスタへの導線を強力なビジュアル（画像）で配置する。
- **メリット**: 最も確実で、CMSの技術的制約を受けない。

## 3. 次世代プラットフォーム（施策2）への検討
現在の Movable Type 環境から「脱出」し、モダンなスタック（Next.js / Vite / WordPress等）に移行する場合の検討項目。

- **独自ドメイン (`toba-hs.jp` 等) の取得**:
    - ブランド力は向上するが、サーバー維持費と管理責任が学校に発生する。
- **ハイブリッド構成の提案**:
    - `kyoto-be.ne.jp` 内の現行サイトは「公式・アーカイブ」として残し、中学生向けの「特設・プレゼンサイト（本プロジェクトで作成中のようなモダンなもの）」を独自ドメインまたは GitHub Pages/Vercel 等で運用する。

## 4. 結論と推奨アクション
1. **短期的対応**: 現行MT環境で、Instagramへのリンク導線をスマホファーストで再設計する。
2. **中期的対応**: `InstaPost` 等の自動記事化が現在の共有MT環境で利用可能か、管理者に問い合わせる。
3. **長期的対応**: 令和9年度入試一本化に向けた「広報特設サイト」を、現行システムとは切り離して独立運用する。
